# 予実管理ダッシュボード（budget-tool）

ワイズ株式会社の予算・実績管理ダッシュボード。Freee会計のCSVデータを取り込み、部門別の経営数字をリアルタイムで可視化する。

## URL

- **アプリ**: https://wiseinc-pharmacy.github.io/budget-tool/
- **リポジトリ**: https://github.com/WISEinc-pharmacy/budget-tool

## 技術スタック

- HTML単一ファイル（index.html）
- Firebase Firestore（データ永続化）
- Firebase Authentication（Google OAuth）
- GitHub Pages（ホスティング）

## 部門一覧（9部門）

| 区分 | 部門 |
|---|---|
| 東葛エリア | 柏、さくら台、我孫子、ながれやま |
| 城北エリア | 足立、アイ調剤 |
| 管理者のみ | 役員、管理部・本部、学術研修企画運営室 |

## 機能一覧

### 予算・実績管理
- 予算入力（初月+年間合計→12か月自動配分）
- 3つの配分パターン（均等割り/均等増減/全月同額）
- 実績入力
- 予実比較タブ
- 年度比較タブ（前年度vs当年度）
- 年度をまたぐ連続期間表示

### Freee CSV取込
- Freee会計の「月次推移：損益計算書」CSVを一括取込
- 部門自動判定（CSV内の物件名・勘定科目ヒントから推定）
- 年度自動検出（CSVタイトルから）
- 部門をドロップダウンで手動変更可能
- プレビュー確認後に実績データに反映
- 編集モード時のみ表示（操作ミス防止）

### マッピング対応科目
技術料、薬剤料、治療器材料、給料手当、法定福利費、福利厚生費、外注費、外注費（フリーランス）、旅費交通費、通信費、消耗品費、車両費（カーシェア/ガソリン/駐車場）、地代家賃、水道光熱費、リース料、減価償却費、役員報酬、租税公課、保険料、支払手数料/支払報酬、本部経費

### その他機能
- CSVエクスポート（予算/実績/全体）
- 権限管理（管理者/編集者/閲覧者）
- Undo/Redo（Ctrl+Z/Y、最大50操作）
- バージョン履歴（Firestore、全部門横断、復元可能）
- KPI表示（技術料労働分配率、技術料労働/設備分配率）
- 営業外収益・費用・経常利益（管理者のみ表示）
- 期間合計列の視覚的強調（ダークブルー背景+白文字）

## Firebase設定

- **プロジェクトID**: wise-budget-26d4c
- **Firestoreルール**: ログインユーザーのみ読み書き可（versionsサブコレクション含む）
- **データ形式**: `FY{年度}_{部門ID}`（例: FY2026_kashiwa）
- **承認済みドメイン**: wiseinc-pharmacy.github.io

## 運用フロー（Freee連携）

1. 税理士がFreeeから店舗別にCSVを出力
2. 麗華（AI秘書Bot）DMにCSVを送信 → `received_files/` に保存
3. budget-toolの「Freee取込」ボタンでCSVを読み込み → 実績データに反映

## 部門自動判定ロジックの改善ポイント

Freee CSV取込時の部門自動判定（`DEPT_HINTS` + フォールバック）について、コードレビューで確認された改善余地。

### 短期（次回取込前に対応推奨）

| # | 問題 | 現状 | 改善案 |
|---|---|---|---|
| 1 | `採用教育費`L3なし時の取りこぼし | L3内訳がないと`kenshu`にも`saiyo`にも計上されない | 本部経費と同様、L3合計=0の場合は全額`kenshu`にフォールバック |
| 2 | `FREEE_SUB_MAP`が dead code | index.html:1605で定義されているが参照箇所なし。L3処理は`parseFreeeCSV`内でインライン | 定数を削除し、インライン処理に統合（ドキュメントとの一致が重要） |
| 3 | `FREEE_DEPT_MAP`が dead code | index.html:1570で定義されているが参照箇所なし | 将来のCSVフォーマット変更（部門名直接出力）に備えて残すか削除するか判断 |

### 中期

| # | 問題 | 改善案 |
|---|---|---|
| 4 | 前月比チェックなし | 取込直後に前月比±30%超の科目を自動警告するモーダルを追加 |
| 5 | 学術研修の判定閾値が固定値 | `totalSales > 2000000`（200万円）は現実的だが、学術研修の規模拡大時に誤判定するリスクあり。閾値を定数化して変更しやすくする |
| 6 | ファイル名補助判定 | 税理士が`足立_2026.csv`等の命名規則を徹底すれば、ファイル名をヒントに使える |

### 長期

| # | 改善案 |
|---|---|
| 7 | 物件名ヒント（`DEPT_HINTS`）の動的管理UI — 店舗移転/新物件追加時にコード修正なしで更新できる管理者設定画面 |
| 8 | Freee API直接連携 — CSVダウンロード・ファイル送信を省略し、月次自動取込スケジューラと組み合わせる |

### 現在の判定精度（2026年度実績）

| 判定精度 | 説明 | 部門数 |
|---|---|---|
| high（L3物件名一致） | 足立・柏・アイ調剤・ながれやま・さくら台・我孫子 | 6 |
| medium（勘定科目構造） | 役員・管理部・本部・学術研修 | 3 |
| none（不明） | 0 | — |

全9部門が自動判定済み（重複ファイル除く）。詳細は `docs/freee_csv_mapping.md` 参照。

## 自動連携に向けての技術的課題

3系統のCSVデータ（患者請求CSV / Freee P/L CSV / budget-tool CSV）のフルパイプライン自動化に向けた課題。
詳細は `docs/format_comparison.md` を参照。

### 短期（現在の運用改善）

| # | 課題 | 現状 | 対応方針 |
|---|------|------|---------|
| 1 | Freee CSVの手動アップロード | UIから毎回ファイル選択が必要 | Node.js + Firebase Admin SDKでCLI取込スクリプト化 |
| 2 | 採用教育費L3「未選択」の振り分け誤り | 全額kenshuに入る（採用費の可能性） | Freee側でL3補助科目入力を徹底 |
| 3 | 重複ファイル手動除外（さくら台・我孫子） | プレビューで警告表示のみ | コンテンツハッシュによる自動スキップ（改善案B） |
| 4 | 取込後の前月比チェックがない | 異常値でも気づかない | 取込直後に±30%超の科目を警告表示 |

### 中期（自動化パイプライン構築）

| # | 課題 | 内容 | 技術的ポイント |
|---|------|------|--------------|
| 5 | Freee CSV受信〜Firestore反映の自動化 | 麗華が受信後、人手を介さずFirestoreに反映 | Node.js + Firebase Admin SDK。部門判定ロジックのCLI化が必要 |
| 6 | 患者請求CSV → Freee売上整合チェック | 患者請求集計値とFreee記録値の照合 | 保険計×部門 = Freee技術料+薬剤料+治療器材料 の自動検算 |
| 7 | 患者請求CSV → patient-db 突合 | 患者IDが空欄の場合の名寄せ | 患者名+カナによる2次突合ロジック（同姓同名対策が必要） |
| 8 | 部門「不明」時の通知 | 新店舗追加や物件名変更時に判定失敗 | DEPT_HINTSの未一致ファイルをSlack通知 |

### 長期（Freee API直接連携）

| # | 課題 | 内容 | 前提条件 |
|---|------|------|---------|
| 9 | Freee API→Firestore自動同期 | 月次でFreee APIをポーリングし、部門別P/Lを自動取込 | Freeeアプリ登録 + OAuth設定 |
| 10 | 患者請求集計の完全自動化 | 調剤システムからCSV自動出力 + Freee仕訳自動生成 | 調剤システムAPI連携（ベンダー交渉が必要） |
| 11 | DEPT_HINTSの動的管理 | 物件名ヒントを管理者UIから更新（コード修正不要に） | budget-tool管理者設定画面の追加 |

### データフォーマット上の固有課題

| 課題 | 詳細 | 対応策 |
|------|------|--------|
| 患者CSVの患者ID欠落 | Col 21（患者ID）が空欄の場合がある | 患者名+カナの組み合わせで代替突合 |
| 患者CSVのカナ列ヘッダー未設定 | Col 23のヘッダーが空欄だがデータはある | パーサーで列番号23を固定参照 |
| Freee売上原価の期初一括計上 | 棚卸差引が2月のみに出る（他月0） | 売上比例配分オプション（改善案として提案中） |
| Freee P/L部門情報なし | CSVにdept列がなく補助科目名で推定 | DEPT_HINTSのメンテナンスが永続的に必要 |
| Freee賞与の不規則出現 | 2025FYのみ賞与が出現。2026FYは0 | 毎年確認が必要。Freeeへの入力タイミング依存 |
