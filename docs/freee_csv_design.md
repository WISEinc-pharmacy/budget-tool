# Freee CSV → budget-tool 設計ドキュメント

最終更新: 2026-04-19

---

## 1. Freee CSVフォーマット仕様

### 1.1 ファイル仕様

| 項目 | 内容 |
|---|---|
| 出力元 | Freee会計 > 月次推移：損益計算書 |
| 文字コード | Shift-JIS（麗華で UTF-8 変換済み） |
| 区切り文字 | カンマ区切り、全フィールドダブルクォート囲み |
| 行数 | 約50〜100行（部門によって異なる） |

### 1.2 カラム構造（20列）

```
Col 0: L0（大項目 / 集計行ラベル）
Col 1: L1（中項目）
Col 2: L2（勘定科目）
Col 3: L3（補助科目）
Col 4: L4（補助科目2）
Col 5: L5（部門または未選択）
Col 6: 空白
Col 7〜18: 月別金額（2月〜1月、12ヶ月分）
Col 19: 期間累計
```

### 1.3 行の種類と判定ルール

| 行タイプ | L0 | L1 | L2 | L3 | 内容 |
|---|---|---|---|---|---|
| セクション開始 | `売上高` | 空 | 空 | 空 | 値なし |
| 売上合計 | 空 | `売上高` | 空 | 空 | 売上合計金額 |
| 売上内訳 | 空 | 空 | `技術料`等 | 空 | 売上科目別 |
| 売上原価 | 空 | 空 | `商品売上原価` | 空 | 原価合計 |
| 販管費開始 | 空 | `販売管理費` | 空 | 空 | 値なし |
| 販管費科目 | 空 | 空 | `給料手当`等 | 空 | 科目合計 |
| 販管費内訳 | 空 | 空 | 空 | 補助科目名 | 科目内訳 |
| 集計行 | `売上総損益金額`等 | 空 | 空 | 空 | Freeeが自動計算 |

### 1.4 年度検出

- 行1のタイトルから `期間[：:](\d{4})年` で年度を自動抽出
- 例: `期間：2026年02月～2027年01月` → FY 2026

### 1.5 月インデックス変換

- Freeeは暦年月（2026-02 等）で出力
- budget-toolは2月始まり: 2月=index0、3月=index1、...、1月=index11
- 変換式: `idx = (month - 2 + 12) % 12`

## 3. 部門自動判定ロジック

### 3.1 物件名ヒント（DEPT_HINTS）

2026-04-19変更: L3物件名（高精度）と全文テキスト（低精度）の2段階判定に変更。

| 部門 | ID | L3物件名ヒント（高精度・`confidence: high`） | 全文テキストヒント（低精度・`confidence: medium`） |
|---|---|---|---|
| 足立 | `adachi` | 中村ビル、伊藤駐車場、足立NSヒタチ、モリタ（足立 | 足立NS |
| 柏 | `kashiwa` | サンライズイシド、セリザワ ヒデオ、柏CLC、宇田川駐車場 | （なし） |
| ながれやま | `nagareyama` | ヒルフォレスト、南流山 | ながれやま |
| さくら台 | `sakuradai` | 千修ビル、鹿山駐車場、鹿山 | さくら台 |
| 我孫子 | `abiko` | オカモトフミコ、オカモト フミコ | 我孫子、あびこ |
| アイ調剤薬局 | `ai_chozai` | 幸楽土地、㈱幸楽 | アイ調剤 |

判定の優先順位:
1. L3列（補助科目）に含まれるテキストとL3ヒントが一致 → `confidence: high`
2. CSV全文テキストにテキストヒントが含まれる → `confidence: medium`
3. いずれもヒットしない → フォールバック判定（Section 3.2）→ `confidence: medium` or なし

削除されたパターン（誤判定リスクのため）:
- `sakura`（「桜」を含む無関係テキストに誤判定）
- `流山`（単独では「南流山」と区別できないため `南流山` に統合）

### 3.2 フォールバック判定

物件名ヒントで判定できない場合のルール:

```
1. 売上0 + 技術料なし
   ├─ 役員報酬あり + 給料手当なし → 役員 (yakuin)
   └─ それ以外 → 管理部・本部 (kanribu)

2. 技術料なし + 売上あり
   ├─ 役員報酬あり OR 売上 > 200万 → 役員 (yakuin)
   └─ それ以外 → 学術研修 (gakujutsu)

3. 技術料あり + 薬剤料あり + ヒントなし → 「不明（調剤薬局系）」でUIに警告

4. 上記いずれも該当しない → 「不明(売上金額)」でUIに警告
```

## 4. CSV読み込み機能強化案

### 4.1 実装済み改善一覧

| 課題 | 内容 | 対処 |
|---|---|---|
| 地代家賃 tenpo/shatak未分離 | 全額をtenpoに入れていた | L3で社宅系を判定して分離 ✅ |
| リース料 lease_office/lease_car未分離 | 全額をlease_officeに入れていた | L3で車両系を判定して分離 ✅ |
| 採用教育費 saiyo=0 | L3で研修/採用を振り分けていなかった | L3で採用/研修を振り分け ✅ |
| 本部経費 honbu_yakuin=0 | 全額をhonbu_somuに入れていた | L3で役員/総務を振り分け ✅ |
| ながれやま・アイ調剤ヒント不足 | 物件名ヒントが未登録 | ヒント追加 ✅ |
| 重複ファイル無警告 | 同部門を2回取り込む危険 | 重複検出と警告表示 ✅ |
| フォールバック判定精度低 | 役員/学術/管理部の判定ロジックが弱い | 条件を改善 ✅ |
| 部門判定が全文スキャン（偽陽性リスク） | csvFullText.includes()で全列を検索 | L3列を専用抽出して物件名マッチング ✅ 2026-04-19 |
| `sakura`パターンが汎用的すぎ | 他文書に「桜」があると誤判定の恐れ | 削除し千修ビル/鹿山で判定 ✅ 2026-04-19 |
| 判定信頼度が表示されない | 自動判定が正しいかUI上で確認不能 | `confidence`フィールド追加、プレビューに色付き表示 ✅ 2026-04-19 |
| 未マッピング勘定科目が無警告 | 新規科目がサイレントに抜け落ちる | `unmappedL2`収集、プレビューに警告表示 ✅ 2026-04-19 |
| 足立ヒントが地代家賃のみ | リース料L3「足立NSヒタチ」が未活用 | `足立NSヒタチ`をL3ヒントに追加 ✅ 2026-04-19 |

### 4.2 今後の改善案（v3）

#### 4.2.1 自動処理パイプライン
```
税理士がFreeeからCSV出力
→ 麗華に送信
→ received_files/に保存（Shift-JIS→UTF-8自動変換済み）
→ 自動スクリプトがbudget-toolのFirestoreに直接書き込み（将来）
```

現状はステップ4が手動（ブラウザでUIから取込）。将来的にはNode.jsスクリプトまたはFirebase Functionsで自動化。

#### 4.2.2 Freee API直接連携（長期）
- Freee公式APIを使い、CSVダウンロードを省略
- 月次自動取込スケジューラと組み合わせる
- 要: Freeeアカウントのアプリ登録とOAuth設定

#### 4.2.3 前月比・前年同月比の自動チェック
- 取込直後に前月比 ±30% を超える科目を警告表示
- 異常値の場合は確認ダイアログを出す

## 5. 注意事項

- **賞与（shoyo）**: Freee側では補助科目で入力されるため、CSVに出力されない。手動入力が必要。
- **社宅家賃（shatak）と地代家賃（tenpo）**: L3補助科目の命名に依存するため、Freee側の補助科目設定が重要。
- **リース料の車両/事務機器分離**: 補助科目名に「車」が含まれるかどうかで判定。「NSヒタチ」等のレンタル品は事務機器扱い。
- **本部経費按分**: 店舗CSVに `本部経費` として含まれている場合は按分後の金額。全社CSVには全体額が入る。
- **重複ファイル**: (5)(6)が同一、(10)(11)が同一であることを確認済み。取込時はどちらか一方のみ使用すること。
