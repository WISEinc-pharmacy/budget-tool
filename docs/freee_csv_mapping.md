## 1. Freee CSVフォーマット仕様

### 1.1 ファイル基本情報

| 項目 | 仕様 |
|------|------|
| 文字コード | Shift-JIS（麗華側でUTF-8自動変換済み） |
| 改行コード | CR+LF または LF |
| 区切り文字 | カンマ（,） |
| 引用符 | ダブルクォート（"） |
| 識別文字列 | 1行目に「月次推移」を含む |

### 1.2 行構造

| 行番号 | 内容 | 例 |
|--------|------|----|
| 1行目 | タイトル行 | `"月次推移：損益計算書_ワイズ株式会社（期間：2026年02月～2027年01月、表示単位：円）"` |
| 2行目 | ヘッダー行（月列） | `"","","","","","","","2026-02","2026-03",...,"2027-01","期間累計"` |
| 3行目〜 | データ行 | 最大6列の科目階層 + 月別金額列 + 累計列 |

### 1.3 列構造

| 列番号 | 識別名 | 内容 | 例 |
|--------|--------|------|-----|
| 0 | L0 | 最上位区分（合計行） | `売上高 計`, `営業損益金額` |
| 1 | L1 | 大分類 | `売上高`, `販売管理費`, `営業外収益` |
| 2 | L2 | 勘定科目（主要マッピング対象） | `技術料`, `給料手当`, `車両費` |
| 3 | L3 | 補助科目（内訳） | `駐車場代`, `ガソリン代`, `中村ビル` |
| 4 | L4 | 細分類 | `仕入高`, `未選択` |
| 5 | L5 | 最細分類 | `未選択` |
| 6 | (空白) | 常に空白 | |
| 7〜18 | 月別金額 | ヘッダーで年月を確認（YYYY-MM形式） | `28525810` |
| 最終列 | 期間累計 | 全月合計 | `59705437` |

### 1.4 データ型一覧

| 列 | CSV上の型 | 取込後の型 | 備考 |
|----|----------|-----------|------|
| L0〜L5 | 文字列（引用符付き） | string | 空欄は空文字 `""` |
| 空白列（列6） | 空文字列 | — | 常に空。データなし |
| 月別金額 | 文字列（引用符付き） | parseInt | 円単位整数。`"0"` は0 |
| 期間累計 | 文字列（引用符付き） | parseInt | 全月合計（取込では使用せず） |

> **注意**: 金額は整数（円単位）のみ。小数点なし。マイナス値（費用超過時）も出現する。

### 1.5 年度・月インデックス対応

budget-toolは2月始まり会計年度を採用。

| Freee月 | budget-tool monthIdx |
|---------|---------------------|
| 2月 | 0 |
| 3月 | 1 |
| 4月 | 2 |
| 5月 | 3 |
| 6月 | 4 |
| 7月 | 5 |
| 8月 | 6 |
| 9月 | 7 |
| 10月 | 8 |
| 11月 | 9 |
| 12月 | 10 |
| 1月 | 11 |

計算式: `monthIdx = (month - 2 + 12) % 12`

> - 支払手数料L3に `リブラ`（薬局システム）が出現（2026FY足立）
> - 最後の損益合計行（`営業損益金額` 等）はL0に出力される。取込では無視してよい

## 7. 部門自動判定ルール

### 7.2 フォールバック判定（ヒントで判定不能な場合）

| 条件 | 推定部門 |
|------|---------|
| 売上高=0 かつ 技術料なし かつ 役員報酬あり かつ 給料手当なし | 役員 |
| 売上高=0 かつ 技術料なし かつ 上記以外 | 管理部・本部 |
| 売上高>0 かつ 技術料なし かつ (役員報酬あり または 売上>200万) | 役員 |
| 売上高>0 かつ 技術料なし かつ 上記以外 | 学術研修 |
| 技術料あり かつ 薬剤料あり | 不明（調剤薬局系）→ 手動確認 |

## 11. 実装上の注意事項

1. **Shift-JIS対応**: ブラウザUIでのアップロード時、文字化け（\uFFFD）が5文字超の場合は自動的にShift-JISで再読み込み
2. **重複ファイル**: 2026年度の(6)さくら台と(11)我孫子は重複のためアップロードしない
3. **賞与の扱い**: Freeeに賞与科目がある場合は自動取込（2025FY確認済み）。2026FY以降も同様。Freee未記録の場合は手動入力
4. **本部経費L3なし**: L3内訳がない場合、全額がhonbu_somu（総務経費）に入る。手動確認・修正を推奨
5. **値が0の月**: 取込時は0値は上書きしない（`if (vals[m] !== 0) store[itemId][m] = vals[m]`）
6. **部門選択**: プレビュー画面でドロップダウンから部門を変更可能。自動判定が「不明」の場合は必ず手動選択
7. **全社CSV（部門フィルタなし）**: L3に部門名（柏、足立等）が出現する。物件名ヒントなし→「不明」判定でスキップ推奨。全社CSVを取り込むと誤集計の原因になる。
8. **修繕費（2025FY）**: FREEE_ACCOUNT_MAP に追加済み（→ sga_other）。2026FYには未出現だが今後出る可能性あり。

### 15.2 改善案A: 全社CSV自動検出ロジック

**現状**: 全社CSVは物件名ヒントがないため「不明」判定になるが、検出が消極的（単に「不明」を返すだけ）。

**改善案**: L3に複数の店舗部門名が含まれていれば「全社CSV」と積極的に判定し、アップロードをブロックする。

```javascript
// 全社CSV検出: 複数部門名がL3に出現する場合はブロック
function detectAllDeptCsv(l3Values) {
  const DEPT_NAMES = ['柏', '足立', 'さくら台', 'ながれやま', '我孫子', 'アイ調剤', '学術研修', '総務'];
  const matched = DEPT_NAMES.filter(name => l3Values.some(v => v.includes(name)));
  return matched.length >= 2;  // 2部門以上出現 → 全社CSV
}

// detectDept関数の冒頭に追加:
if (detectAllDeptCsv(l3Values)) {
  return { dept: '全社CSV（取込不可）', deptId: null, confidence: 'all_dept_block' };
}
```

**期待効果**: 全社CSVを誤って取り込むミスをUIレベルで防止。

### 15.3 改善案B: 重複ファイル自動検出

**現状**: (6)さくら台と(11)我孫子が重複するが、手動で除外が必要。

**改善案**: 複数ファイルアップロード時にコンテンツハッシュを計算し、一致するファイルに警告を表示。

```javascript
// ファイルコンテンツのハッシュ計算
async function calcHash(text) {
  const enc = new TextEncoder();
  const buf = await crypto.subtle.digest('SHA-256', enc.encode(text));
  return Array.from(new Uint8Array(buf)).map(b => b.toString(16).padStart(2,'0')).join('');
}

// 複数ファイル解析ループ内で:
const hashMap = new Map();  // hash → ファイル名
for (const file of files) {
  const text = await readFileAsText(file);
  const hash = await calcHash(text);
  if (hashMap.has(hash)) {
    showWarning(`重複ファイル検出: ${file.name} は ${hashMap.get(hash)} と同一内容です（取込をスキップします）`);
    continue;  // スキップ
  }
  hashMap.set(hash, file.name);
  // 以降の解析処理へ...
}
```

**期待効果**: 重複ファイルを自動スキップし、手動確認が不要になる。

### 15.6 改善案E: L3新パターン自動検出

**現状**: 既知のL3パターン（物件名、補助科目名）は文書化されているが、新しいパターンが出た場合に気づかない。

**改善案**: 既知パターンにマッチしないL3値を「新パターン検出」としてログ出力。

```javascript
// 既知L3パターン（参照用）
const KNOWN_L3_PATTERNS = {
  '車両費': ['タイムズカーシェア', 'ガソリン代', 'ガソリン', '駐車場代', '高速代', 'タクシー', 'タクシー代', 'レンタカー代', '未選択'],

  'リース料': ['車', '足立NSヒタチ', '柏CLC', '柏シヤ-プフアイナンス', 'プリンター', '監査支援システム', '未選択'],
  '本部経費': ['会社一括支払分', '本部運営費', 'その他', '未選択'],
};

function detectNewL3(l2, l3Val) {
  const known = KNOWN_L3_PATTERNS[l2] || [];
  const isKnown = known.some(p => l3Val.includes(p));
  if (!isKnown && l3Val && l3Val !== '未選択') {
    console.info(`[Freee取込] 新L3パターン検出: ${l2} > "${l3Val}" (マッピングルール確認推奨)`);
  }
}
```

**期待効果**: 年度変わりで新しい補助科目が出た際に開発者がすぐ気づける。
