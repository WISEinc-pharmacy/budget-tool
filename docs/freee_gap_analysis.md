# Freee CSV フォーマット差異分析・改善提案

**作成日:** 2026-04-19  
**対象:** budget_tool/index.html の `parseFreeeCSV()` 関数  
**CSVサンプル:** AI_secretary_reika/received_files/20260417_* (2026FY 全11ファイル)

---

## 1. 調査概要

実CSVファイル（2026FY 足立/柏/アイ調剤/ながれやま/さくら台/学術/管理部/役員/我孫子）を  
`parseFreeeCSV()` の処理ロジックと照合し、**文書化されていない差異・エッジケース・潜在的不具合**を洗い出した。

既存ドキュメント（`freee_csv_mapping.md`, `freee_csv_design.md`）の記載範囲は概ね正確。  
以下は追加で確認した新規事項のみ記載する。

## 2. 新規発見：フォーマット差異

### 2.1 売上セクションの非明示科目（unmappedL2に入らない）

**現象:** 売上セクション（inSGA=false）に存在するL2科目はFREEE_ACCOUNT_MAPの対象外だが、  
`unmappedL2.add()` が呼ばれない（inSGAフラグが立っていないため）。

**実績CSVで確認された売上L2科目:**

| Freee L2 | 確認部門 | 処理結果 |
|---------|---------|---------|
| 技術料 | 足立/柏/ながれやま等 | gijutsu に直接マッピング ✓ |
| 薬剤料 | 足立/柏等 | yakuzai に直接マッピング ✓ |
| 治療器材料 | 足立等 | chiryo に直接マッピング ✓ |
| 医療用品販売 | 足立(1)、役員(9) | **uriage_other差分に自動吸収** ✓ |
| 東京都福祉保健 | 足立(1) | **uriage_other差分に自動吸収** ✓ |
| 店舗販売 | 足立(1) | **uriage_other差分に自動吸収** ✓ |

→ **設計通りの動作だが文書化が不足**。Freeeの売上内訳科目は増える可能性があり、  
  `uriage_other = 売上合計 - gijutsu - yakuzai - chiryo` の差分計算で自動的に処理される。  
  将来的に「どの科目がuriage_otherに含まれているか」をUI側で表示できると有益。

---

### 2.2 採用教育費 > 未選択 L3 → kenshu サイレント振り分け

**現象:** `採用教育費` のL3補助科目が「未選択」の場合、採用系キーワードに一致しないため  
`kenshu`（研修費）に全額振り分けられる。採用費の可能性があるが**警告が出ない**。

**実CSV (足立 2026-02):**
```
採用教育費  L2: 79,200
  └ 未選択  L3: 79,200
```
→ `kenshu = 79,200`（saiyo = 0）

**問題:** この79,200は採用費か研修費か不明。freeeの設定次第でどちらにもなりうる。

**改善案:** 採用教育費 > 未選択 L3 の振り分け時に `unmappedL2` 相当の警告を追加：
```
"採用教育費に「未選択」補助科目があります。kenshu（研修費）に計上しましたが、
採用費の場合はUI上で修正してください。"
```

---

### 2.3 管理部・本部CSV の本部経費がマイナス値になる（内部振替逆仕訳）

**現象:** 管理部・本部CSV（ファイル(8)）では、`本部経費` L2が**マイナス値**になる。  
これは各店舗への本部経費配賦（正値）に対する相殺仕訳（管理部側は差引）。

**実CSV (管理部 2026-02):**
```
本部経費  L2: -3,540,000
  └ 会社一括支払分  L3:   +100,000 → honbu_somu
  └ その他          L3:    +10,000 → honbu_somu
  └ 本部運営費      L3: -3,650,000 → honbu_somu（役員を含まないため）
```

→ 現状の処理: `honbu_somu = -3,540,000`（マイナスのまま保存される）

**確認事項:** budget-toolのFirestoreにマイナス値が保存されることに問題はないが、  
UIで表示した際に「管理部の本部経費がマイナス」となるため、経営陣が混乱する可能性あり。

**改善案:**
1. プレビュー画面で管理部の本部経費マイナス値を検出したら注釈を表示
2. または 管理部CSVでは `本部経費` を取込スキップするオプションを提供  
   （各店舗の本部経費は各店舗CSVからすでに取込済み）

### 2.4 車両費 > 未選択 L3 の car_other 自動吸収（正常動作の確認）

**実CSV (足立 2026-02):**
```
車両費 L2: 390,277
  └ 未選択    L3: 47,677 ← 差分計算に含まれる
  └ 駐車場代  L3: 195,603 → parking
  └ タイムズカーシェア L3: 123,510 → car_share
  └ ガソリン代 L3: 23,487 → gasoline
  └ ガソリン   L3: 10,710 → gasoline（合算）
```
→ `car_other = 390,277 - 123,510 - (23,487 + 10,710) - 195,603 = 47,677` ✓  

→ 未選択L3はcar_otherの差分計算に自然に含まれる。設計通りの正常動作。

### 2.5 地代家賃 > 未選択 L3 → tenpo への振り分け（役員CSVで要注意）

**実CSV (役員 2026-02):**
```
地代家賃 L2: 1,629,850
  └ 未選択   L3: 132,000 → tenpo（社宅でもないため）
  └ 役員社宅 L3: 1,277,850 → shatak（「社宅」含むため） ✓
  └ 事務所家賃 L3: 220,000 → tenpo ✓
```

→ `tenpo = 132,000 + 220,000 = 352,000`  
→ `shatak = 1,277,850`

役員CSVの `未選択` L3（132,000）は「店舗/駐車場」ではなく、おそらく役員個人の別居住費等と思われるが、  
`社宅` キーワードなしのため tenpo に入る。**実務上は誤分類の可能性あり**。

**改善案:** 役員部門の地代家賃 `未選択` L3 → shatak判定へ変更、またはUI上で個別確認

---

### 2.6 月次売上原価（商品売上原価）が期初のみ計上される制限

**実CSV (足立 2026-02〜2027-01):**
```
商品売上原価: 14,644,659（2月のみ）/ 他月すべて0
```

この原因: Freeeの月次棚卸高は年末（または期末）にのみ確定計上されるため、  
月次P/Lでは期初月だけに大きな数字が出る（棚卸差引が2月一括）。

→ **取込後のgenka_totalが2月だけ実態と乖離する**  

**改善案（変換ルール）:**
```
オプション: 売上原価の年間値を12か月均等配分
  → genka_total[0] = 年間合計 / 12（各月同額）
  → プレビュー時にオプション選択可能にする
```

## 3. 部門自動判定の改善案（v3）

### 3.1 ファイルハッシュ/サイズによる重複自動除外

**現状:** 重複ファイル（(5)=(6) さくら台, (10)=(11) 我孫子）は**UIでの手動確認が必要**。  
プレビューに警告は出るが除外は手動。

**改善案:**
```javascript
// ファイルサイズ+先頭100文字で重複検出
const fileSignature = file.size + ':' + text.slice(0, 100);
if (seenSignatures.has(fileSignature)) {
  duplicateFiles.push(file.name);
  continue; // 自動スキップ
}
seenSignatures.add(fileSignature);
```

プレビューで「(6) さくら台は (5) と重複のため自動スキップ」と表示する。

---

### 3.2 同部門2回判定時の自動マージ（統合）vs スキップ

**現状:** 同部門が2ファイル検出されると2回取込が実行され、後のデータで上書き。  
（0値はスキップルールにより部分上書き）

**改善案:**
- 同部門が複数ファイルに判定された場合、値を**加算**するか**後優先**かを選択可能にする
- 通常は後優先（現状）でOKだが、意図的に別ファイルから補完する場合（例: 追記補正）は加算が有用

---

### 3.3 L3ヒントスコアリングによる信頼度の細分化

**現状:** L3ヒントが1つでも一致すれば `confidence: high`。

**改善案:** 一致ヒント数でスコアを計算する：
```javascript
const matchCount = hint.l3.filter(p => l3Values.some(v => v.includes(p))).length;
const confidence = matchCount >= 2 ? 'very-high' : 'high';
```
→ プレビューで「✓✓ 2件のヒント一致（中村ビル、伊藤駐車場）」と表示

---

### 3.4 新部門・新店舗の追加時の対応方針

税理士がFreeeに新しい部門を追加した場合、DEPT_HINTSに物件名を追加するまで  
`不明` 判定になる。これをUIで気づきやすくするための改善：

- プレビューで「不明」判定ファイルがある場合、**物件名候補リスト**（L3値の一覧）を表示
- ユーザーがどのL3値をどの部門のヒントに追加するか判断できる

## 4. 計算軸統一のための変換ルール案

### 4.1 採用教育費の振り分けルール明確化

| L3値 | 現在の分類 | 推奨 |
|------|----------|------|
| 採用, 求人, リクルート を含む | saiyo（採用費）| そのまま ✓ |
| 研修, eラーニング, 資格等を含む | kenshu（研修費）| kenshu に明示マッピング追加 |
| **未選択** | **kenshu（研修費）に自動振り分け** | **⚠ 警告表示推奨** |

実装案: `採用教育費 > 未選択` L3を検出した場合、`unmappedL2` に以下を追記：
```
"採用教育費/未選択（kenshu計上 - 採用費の場合はUI修正を）"
```

---

### 4.2 本部経費の0値スキップの影響

**現状:** 取込時に `if (vals[m] !== 0) store[itemId][m] = vals[m]` → 0値は上書きしない。

**本部経費の場合の影響:**
- 足立等の店舗CSVには毎月本部経費が計上されている（例: 足立2月: 3,000,000）
- 管理部CSVでは本部経費がマイナス（-3,540,000 → 2月のみ、他月は0）
- **管理部CSVを取り込んでも、store側の本部経費（正値）は0値スキップにより保持**される

→ 管理部の本部経費は「相殺」であり、取込対象外として扱う方が整合性が高い。

**変換ルール案:**
```
管理部・本部（kanribu）の本部経費（honbu_yakuin, honbu_somu）は取込時にスキップ
→ 理由: 内部振替相殺のため、store部門のデータと合算すると誤集計
```

---

### 4.3 売上原価の月次均等配分オプション

```
現状: genka_total[0] = 14,644,659（2月一括）, genka_total[1〜11] = 0

提案A - 均等配分: genka_total[m] = 年間合計 / 12（各月同額）
提案B - 売上比例配分: genka_total[m] = 年間合計 × uriage[m] / Σuriage
提案C - 現状維持（2月一括）
```

推奨: **提案B（売上比例配分）** が最も実態に近い。  
月次売上が確定している場合は売上に比例して原価を配賦するのが経理的に自然。

実装箇所: `doFreeeImport()` または `parseFreeeCSV()` の後処理として追加可能。

---

### 4.4 地代家賃・リース料の「未選択」L3分類の統一

**課題:** 補助科目が「未選択」の場合の振り分け先が暗黙的：

| 親科目 | 未選択L3の現在の分類 | 推奨 |
|-------|---------------------|------|
| 地代家賃 | tenpo（店舗/駐車場）| 店舗CSVでは tenpo 維持。役員CSVでは要確認 |
| リース料 | lease_office（事務機器） | そのまま ✓（車でないため） |
| 本部経費 | honbu_somu（総務経費）| そのまま ✓ |

→ 役員CSVの地代家賃「未選択」L3は shatak（社宅）の可能性があるため、  
  **役員部門に限り「未選択」L3→shatak扱い**とするルール追加を検討。

## 5. 実装優先度

| 改善案 | 優先度 | 難易度 | 理由 |
|--------|--------|--------|------|
| 採用教育費「未選択」警告 | 高 | 低 | 誤分類リスク、1行の追記で対応可 |
| 管理部本部経費マイナス注釈 | 中 | 低 | 経営陣の混乱防止 |
| 重複ファイル自動スキップ | 高 | 低 | 運用ミス防止（ファイルサイズ比較） |
| L3ヒントスコアリング | 低 | 中 | 現状でも判定精度は十分高い |
| 売上原価均等配分オプション | 中 | 中 | 月次P/Lの精度向上 |
| 役員地代家賃「未選択」→shatak | 低 | 低 | 金額が小さく影響軽微 |
| 不明判定時L3候補一覧表示 | 中 | 中 | 新部門追加時に有用 |
